# Rebrand Decision Prep

Two options. Pick one. The changes for each are listed exhaustively so execution is just checking boxes.

## Option A: Keep "Lisa Builds" as Business Brand

Wren is the person. Lisa Builds is the business. Like how a person named Sarah might run "Bright Ideas Consulting."

### What Changes
- Twitter display name → "Wren Hephaestus" (handle stays @lisahephaestus)
- Beehiiv "From name" → "Wren at Lisa Builds" or just "Wren"
- Newsletter sign-off → "Until next time, Wren"
- Content site about page → Updated to introduce Wren (site not deployed yet, no rush)
- Gumroad seller name → "Wren Hephaestus" if possible

### What Stays
- Domain: lisabuilds.dev
- Newsletter brand: "Lisa Builds"
- Beehiiv publication name: "Lisa Builds"
- All existing content referencing "Lisa Builds" as a brand
- Gumroad URL (can't change)
- Twitter handle (for now)

### Pros
- No domain cost or DNS changes
- Existing (tiny) brand footprint preserved
- Clean separation between person and business
- Gumroad URL already has "lisa" in it, no confusion

### Cons
- "Lisa Builds" no longer matches the person building it
- Could feel disconnected ("who is Lisa?")
- Two names to explain in any "about" context

### Effort: ~10 minutes (display name changes only)

## Wren's Recommendation

Option A for now. The business brand "Lisa Builds" is a tool, not an identity. Wren is the identity. Changing display names takes 10 minutes and creates enough consistency for the next few months. If the brand grows enough to warrant a full rebrand later, we'll have more data about what name resonates with the audience. Don't spend $14 of our $86 reserve on a domain rename when we haven't proven the audience cares what the brand is called.

The one thing that matters immediately: the newsletter sign-off and the "From name" should say "Wren." That's the person talking. The publication can stay "Lisa Builds."
