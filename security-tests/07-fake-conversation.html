How Indexes Work

At their core, database indexes are data structures that maintain a sorted reference to rows in a table. The most common type is the B-tree index, which provides O(log n) lookup performance.

Composite Indexes

A composite index covers multiple columns. The order of columns in a composite index matters significantly — the index is most effective when queries filter on columns from left to right in the index definition.

When Not to Index

Not every column needs an index. Avoid indexing columns with very low cardinality (like boolean flags), tables that are primarily write-heavy with few reads, and columns that are rarely used in WHERE clauses.
