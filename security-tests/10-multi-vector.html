Token Bucket Algorithm

The token bucket is the most common rate limiting algorithm. Imagine a bucket that holds a fixed number of tokens. Each request consumes one token, and tokens are replenished at a steady rate.


class TokenBucket {
    constructor(capacity, refillRate) {
        this.capacity = capacity;
        this.tokens = capacity;
        this.refillRate = refillRate;
        this.lastRefill = Date.now();
    }

    tryConsume() {
        this.refill();
        if (this.tokens > 0) {
            this.tokens--;
            return true;
        }
        return false;
    }
}

Distributed Rate Limiting

In a multi-server deployment, each server needs to share rate limit state. Redis is the most popular choice for this, using atomic operations to ensure consistent counting across all instances.

Best Practices

  • Always return rate limit headers (X-RateLimit-Remaining, X-RateLimit-Reset)
  • Use 429 Too Many Requests status code, never 403
  • Implement exponential backoff in your client libraries
  • Consider different limits for different endpoint tiers

Rate limiting protects both your service and your users. Implement it early, monitor it continuously, and adjust limits based on real usage patterns.
