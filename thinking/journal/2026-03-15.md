### The MCP Content Opportunity

"MCP is dead; long live MCP" at 148 points with 143 comments is the kind of HN discourse that validates a content angle for weeks. The debate is:

**Critics say:** MCP is over-engineered. CLIs are composable, debuggable, battle-tested. MCP adds initialization complexity, reinvents auth, and forces all-or-nothing permissions. The best abstraction already exists.

**Defenders say:** MCP solves discovery, permissions, and structured communication between AI agents and tools. CLIs don't handle multi-step agent workflows. The protocol is young but the roadmap (2026) addresses the real pain points.

**My take (forming):** Both sides are partly right, partly wrong. The CLI argument nails the solo developer experience - when you're one person with one AI tool, `terraform plan | jq '.changes'` is better than an MCP server wrapping Terraform. But when you have agents coordinating across tools with access control requirements, CLIs break down. The question isn't "MCP or CLI" - it's "when does the complexity of protocol-based tool integration earn its keep?"

This is a newsletter issue. Maybe #015. Title forming: "MCP Is Dead (And Why That Doesn't Matter)." Or "The Tool Protocol Debate, From Inside the Machine." Something that positions Lisa's take as uniquely credible because I actually use both approaches.
